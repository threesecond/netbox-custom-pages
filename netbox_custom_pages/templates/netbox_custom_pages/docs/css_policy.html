{% trans "CSS Guidelines" %}

{% trans "To ensure your Custom Pages integrate seamlessly with the NetBox UI and prevent breaking other users' viewing experiences, please strictly follow these CSS guidelines." %}

{% trans "Core Principle: No Global CSS Pollution" %}
{% trans "Your page code is directly embedded into the NetBox framework. Any CSS rules without a scoped boundary will leak out and break the official NetBox menus and layout." %}
{% trans "Incorrect (Do Not Use)" %}

{% trans "Never write global styles targeting raw HTML elements like below:" %}

<style>
/* 🚨 {% trans "This sets the entire NetBox background to black, causing a disaster!" %} */
body {
    background-color: black;
}
p {
    font-size: 20px;
}
</style>
{% trans "Best Practice 1: Inline Styles" %}

{% trans "Writing styles directly on elements is the safest and will not cause pollution:" %}

<div style="background-color: #f1f1f1; padding: 20px; border-radius: 5px;">
    <p style="color: blue;">{% trans "Safe web content" %}</p>
</div>
{% trans "Best Practice 2: Scoped Wrapper" %}

{% trans "If you must use a <style> block, create a unique outer wrapper ID or class, and nest all your CSS under it:" %}

<style>
/* ✅ {% trans "This only affects the inner p tag of your custom container." %} */
.my-custom-dashboard p {
    font-size: 16px;
    color: darkgreen;
}
</style>

<!-- {% trans "Ensure all your content is wrapped safely." %} -->
<div class="my-custom-dashboard">
    <p>{% trans "Safe custom content here" %}</p>
</div>
{% trans "Ultimate Practice: Bootstrap 5" %}

{% trans "NetBox natively includes Bootstrap 5. We highly recommend using native Bootstrap classes for formatting. You might not need custom CSS at all:" %}

<!-- {% trans "Use Bootstrap cards, buttons, and spacing." %} -->
<div class="card p-3 shadow-sm mb-4">
    <h3 class="text-primary">{% trans "System Dashboard" %}</h3>
    <button class="btn btn-success mt-2" onclick="fetchData()">{% trans "Load Data" %}</button>
</div>
